# 存檔遷移教學

**繁體中文** | [简体中文](MIGRATION.zh-CN.md)

把既有的 Palworld 世界搬進 palserver GUI。**v2.1 起搬家功能已內建**:匯入存檔、
修復主機角色都是按鈕操作,不用複製檔案、不用跑指令、不用裝 Python。

> **開始之前**:來源端的遊戲 / 伺服器要**完全關閉** —— Palworld 執行時會持續寫入存檔,
> 複製到一半的檔案是壞的。GUI 在匯入與修復前都會**自動備份**,出問題可一鍵還原。

---

## 內建流程(建議)

### 第 1 步:匯入存檔

1. 首頁按「建立伺服器」旁的「**匯入存檔**」。
2. 選來源類型(只影響提示文字,三種的處理方式相同):
   | 來源 | 存檔在哪 |
   | --- | --- |
   | **其他專用伺服器** | 舊伺服器目錄(或其中的存檔資料夾),整個目錄貼上來就行 |
   | **本機共玩存檔**(四人邀請碼) | `%LOCALAPPDATA%\Pal\Saved\SaveGames\<SteamID>\` |
   | **舊版 1.0 GUI** | 在 v1 介面按「開啟伺服器資料夾」看到的目錄 |
3. 把資料夾路徑貼上(存檔資料夾或整個伺服器目錄都可以)→「**掃描**」→ 從找到的世界中選一個。
4. 「下一步:設定伺服器」→ 填伺服器名稱等 →「**建立並匯入**」。
   世界會自動帶入新伺服器並設為啟用,啟動即可繼續玩。

> **v1 在同一台機器?** 更省事:「建立伺服器」時把「既有伺服器路徑」填成 v1 的伺服器目錄,
> 原地接管,連搬都不用搬。

### 第 2 步(僅本機共玩存檔需要):修復主機角色

共玩存檔的**主機玩家**角色綁在通用 ID 上,直接進專用伺服器會被要求重建角色
(帕魯與建築不受影響)。GUI 已內建修復工具(等同社群的 palworld-host-save-fix):

1. **啟動伺服器**,主機玩家用自己的帳號**加入一次**(會產生一個新的空角色檔)。
2. **停止伺服器**。
3. 到「**存檔備份**」分頁 —— 偵測到共玩主機檔的世界會出現「**修復主機角色**」按鈕。
4. 點開後選主機的新角色檔(剛加入產生的那個會標「**匯入後新增**」並自動預選)→「執行修復」。
5. 再啟動伺服器,主機就能用原本的角色進來了。

> - 修復前會強制自動備份整個世界,可在備份清單一鍵還原。
> - 新版遊戲的 **PlM(Oodle)存檔格式**已支援,第一次會自動下載解壓元件。
> - 若修復後**公會**顯示異常:讓該玩家退出公會再重新加入即可(其他玩家不受影響)。

### 遷移後檢查清單

- [ ] 存檔備份分頁裡,你的世界標示為「**啟用中**」(匯入流程會自動設好)
- [ ] 世界大小合理(空世界通常不到 1 MB;玩過的世界幾十 MB 起跳)
- [ ] 啟動後玩家能用原本的角色進入
- [ ] 順手按一次「立即備份」

## 附錄:存檔結構與手動遷移

一般情況用上面的內建流程即可;這節留給想了解原理、或需要手動處理特殊情況的人。

```
<伺服器目錄>/Pal/Saved/
├─ SaveGames/0/<世界 GUID>/          ← 一個世界一個資料夾
│  ├─ Level.sav                      世界本體(建築、帕魯、地形改變)
│  ├─ LevelMeta.sav
│  ├─ WorldOption.sav                建立世界時的設定快照
│  └─ Players/<PlayerUid>.sav        每位玩家的角色
└─ Config/WindowsServer/
   ├─ PalWorldSettings.ini           世界設定(由 GUI 產生,別手動改)
   └─ GameUserSettings.ini           ★ DedicatedServerName 決定載入哪個世界
```

**最重要的一件事**:伺服器不會自動載入「唯一那個」世界資料夾,而是讀
`GameUserSettings.ini` 裡的 `DedicatedServerName=` 去找對應 GUID 的資料夾。
手動遷移失敗九成是這裡沒對上 —— 伺服器找不到,就默默開一個全新世界。
在 GUI 的存檔備份分頁按「設為啟用世界」就會幫你改好這一行。

**手動搬世界**:把整個 `<世界 GUID>` 資料夾放進目標實例的 `SaveGames/0/` 底下
(可用「模組」分頁的檔案管理器上傳),再「設為啟用世界」。
**不要**直接複製舊的 `GameUserSettings.ini` —— 它夾帶舊主機的 `PublicIP`、
`PublicPort`、RCON 設定,只有 `DedicatedServerName` 一行是需要的,而那行 GUI 幫你改。

**手動修主機角色**:內建的「修復主機角色」等同社群工具
[palworld-host-save-fix](https://github.com/xNul/palworld-host-save-fix) 的主路徑
(guild_fix=False);想手動跑的話按該工具的說明操作,記得先備份。

## 常見問題

- **匯入後進來是全新世界?** 檢查存檔備份分頁的「啟用中」標記是否在你的世界上
  (內建流程會自動設定;手動搬才會遇到)。
- **主機以外的玩家也要修復嗎?** 不用。共玩的其他成員與專用伺服器用同一種 ID,
  直接進來就是原本的角色;只有主機綁在通用 ID 上需要過戶。
- **修復壞了怎麼辦?** 修復前有強制自動備份 —— 存檔備份分頁最上面那份,一鍵還原。
