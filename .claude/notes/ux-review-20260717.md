# 雙人設 UX 審視報告(新手服主 × 進階管理者)

> 2026-07-17,8 個評審 agent 並行審查六大功能域+兩個旅程視角,共 83 條發現(全部附檔案:行號)。
> 完整原始資料:scratchpad/ux-audit.json(83 條含 evidence/suggestion/effort)。
> 本檔是彙整後的優先序;「同時滿足兩種人」的原則 = 預設簡單、進階可及(progressive disclosure)。

## 主題一:出事的時候要說人話(新手最痛,涵蓋雙方)

1. **伺服器意外停止無提示**:啟動後閃退(壞 ini/mod 衝突最常見)只是狀態默默跳回「已停止」,新手以為自己沒按到。→ 偵測「啟動後短時間內轉 stopped」顯示醒目橫幅+查看日誌按鈕(InstanceDetail.tsx:124-128)【M】
2. **啟動/停止錯誤是天書**:API 原始錯誤直接進紅色橫幅,無下一步;installError 有白話+日誌連結,一般錯誤沒有(InstanceDetail.tsx:167 vs 341-358)【M】
3. **supervisor 靜默放棄**:崩潰重啟達上限後不再重啟,唯一線索埋在重啟紀錄裡【M】
4. **日誌不可讀**:無錯誤/警告過濾(classifyLine 只拿來上色)、新行強制捲底讓人讀不完錯誤【S×2】
5. **模組下載失敗錯誤英文直出**(HTTP 403 等)【S】

## 主題二:改版日生存包(每次遊戲更新都會發生)

6. **更新遊戲不提模組**:VersionCard 更新確認只講「下載數 GB」,更新後 UE4SS/PalDefender 極可能失效卻無提醒【M】
7. **模組無「有新版」偵測**:marker 記了版本但不比對 GitHub latest【M】
8. **沒有「暫時停用不刪檔」的退路**:改版警告建議「先移除模組」,但移除 UE4SS 連使用者的 Lua 模組一起永久刪除(mods.ts);至少移除前自動備份【M】
9. **版本檢查被動**:只在 VersionCard 掛載時查一次,改版日 GUI 其他地方零提示【M】

## 主題三:建立後的引導鏈(新手)

10. **精靈完成即斷線**:onCreated 丟棄 created.id 只回列表,應直接導向新實例頁+「安裝中→啟動→邀請朋友」進度鏈(App.tsx:604-619)【M】
11. **空狀態無 CTA**:「還沒有伺服器,建立第一個吧!」沒有按鈕【S】
12. **朋友端最後一哩**:三種連線方式都停在「把位址傳給朋友」,沒說朋友要在遊戲哪裡輸入(進遊戲→加入多人→輸入位址)【S】
13. **設定連結過期=死路**:文案叫遠端朋友「重新輸入」配對碼,但配對碼在主人電腦上【S】
14. **web 站沒有「我還沒裝 agent」出口**【S】

## 主題四:處處要搜尋(雙方,同一個模式重複四次)

15. 世界設定 118 項無搜尋(SettingsEditor)【M】
16. PalDefender 45+ 項無搜尋【S】
17. 地圖無玩家/據點搜尋(flyTo 機制已存在)【M】
18. 首頁實例列表無搜尋/篩選(10+ 台時)【S】

## 主題五:進階規模化

19. **範本建立**:精靈加「從現有伺服器帶入設定」/卡片「複製建立」【M】
20. **多連線管理**:只能存一組 agent 連線,切換要重配對【M】
21. **批次操作**:130+ 端點全是單實例,首頁無多選(全部啟動/停止/更新)【M】
22. **設定跨台複製**:世界設定/模組組合/分頁配置無法套用到其他實例【M×3】
23. **API 文件**:agent 有完整 REST API 但零文件,自動化只能逆向原始碼【M】
24. **開機自啟**:整套系統不會活過主機重開機(無 schtasks/服務註冊)【M】
25. **效能歷史**:走勢只存記憶體 5 分鐘,無法回溯「昨晚 FPS」【L】
26. **日誌搜尋/匯出**【M】、**分頁快捷鍵**【S】、**分頁狀態進 URL**【M】

## 主題六:危險操作防呆(新手)

27. **白名單空開鎖全服**:啟用白名單但 GUI 無名單管理介面、無警告【M】
28. **還原備份不敢按**:confirm 不說會回溯多少進度(備份時間 vs 現在)【M】
29. **刪檔級選項無警告**:bHardcore/bPalLost/關 RCON 會斷 GUI 功能【M】
30. **踢出/封鎖用原生 confirm**:無原因欄、不解釋差別;離線玩家無法封鎖(最常見情境)【M+S】
31. **PublicPort 誤導**:標「端口號」但改了不影響實際監聽埠【S】

## 結構級缺口(需要規劃,不是快修)

32. **通知系統**:崩潰/熔斷/改版/備份失敗都沒有主動通知管道(Discord webhook 是服主圈標配)【L,價值最高的結構投資】
33. **權限模型**:單一全權 token 無角色/撤銷/審計,「朋友一條連結加入管理」= 交出全權【L】
34. **備份體驗**:多世界混雜、無下次執行時間、docker 顯示可開實際不跑、lastResult 中文寫死【M 群】
35. **埠管理集中化**:埠設定散落 4 處,建立後無 UI 可改遊戲埠(API 已存在)【M】

## 與既有決策相衝、建議重新考慮的兩項

- **邀請卡的 LAN 位址**:agent 已回報 info.lan 但卡片完全沒用它;「朋友就在同一個 WiFi」是最簡單情境卻被導去裝 playit。先前你把「本機連線」選項拿掉了——評審建議至少在偵測到 LAN 時顯示一個可複製的區網位址 chip(不用做成第四選項)。
- **playit 位址回填**:externalAddress API 還在但 UI 已無入口;朋友要的「一鍵複製的專屬位址」現在永遠不會出現在卡上。先前你判斷貼位址「沒意義」——評審的角度是「設定完成後的位址展示」而非設定過程。
