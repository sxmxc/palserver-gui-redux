# 存档迁移教程

[繁體中文](MIGRATION.md) | **简体中文**

将现有 Palworld 世界迁移到 palserver GUI。**从 v2.1 开始,迁移功能已内置**:导入存档、
修复主机角色均可通过界面操作,无需手动复制文件、运行命令或安装 Python。

> **开始之前**:来源端的游戏 / 服务器必须**完全关闭**。Palworld 运行时会持续写入存档,
> 复制到一半的文件会损坏。GUI 会在导入和修复前**自动备份**,出现问题时可一键恢复。

---

## 内置流程(推荐)

### 第 1 步:导入存档

1. 在首页点击“创建服务器”旁的“**导入存档**”。
2. 选择来源类型(只会影响提示文字,三种来源的处理方式相同):

   | 来源 | 存档位置 |
   | --- | --- |
   | **其他专用服务器** | 旧服务器目录或其中的存档文件夹,直接粘贴整个目录路径即可 |
   | **本机联机存档**(四人邀请码) | `%LOCALAPPDATA%\Pal\Saved\SaveGames\<SteamID>\` |
   | **旧版 1.0 GUI** | 在 v1 界面点击“打开服务器文件夹”后显示的目录 |

3. 粘贴文件夹路径(可以是存档文件夹或整个服务器目录)→“**扫描**”→ 从扫描结果中选择一个世界。
4. 点击“下一步:设置服务器”→ 填写服务器名称等信息 →“**创建并导入**”。
   世界会自动导入新服务器并设为当前世界,启动后即可继续游玩。

> **v1 位于同一台机器?** 可以使用更简单的方法:创建服务器时,将“现有服务器路径”设置为
> v1 的服务器目录,即可原地接管,无需迁移文件。

### 第 2 步(仅本机联机存档需要):修复主机角色

联机存档的**主机玩家**角色绑定在通用 ID 上,直接进入专用服务器时会被要求重建角色
(帕鲁与建筑不受影响)。GUI 已内置修复工具,作用等同于社区工具 palworld-host-save-fix:

1. **启动服务器**,让主机玩家使用自己的账号**加入一次**(会生成一个新的空角色文件)。
2. **停止服务器**。
3. 打开“**存档备份**”标签页。检测到联机主机角色文件的世界会显示“**修复主机角色**”按钮。
4. 选择主机玩家的新角色文件(刚加入后生成的文件会标记为“**导入后新增**”并自动选中),
   然后点击“执行修复”。
5. 再次启动服务器,主机玩家即可使用原来的角色进入游戏。

> - 修复前会强制自动备份整个世界,可在备份列表中一键恢复。
> - 已支持新版游戏的 **PlM(Oodle)存档格式**,首次使用时会自动下载并解压所需组件。
> - 如果修复后**公会**显示异常,让该玩家退出公会后重新加入即可,其他玩家不受影响。

### 迁移后检查清单

- [ ] 在“存档备份”标签页中,你的世界标记为“**当前世界**”(导入流程会自动设置)
- [ ] 世界大小合理(空世界通常小于 1 MB,游玩过的世界一般为几十 MB 或更大)
- [ ] 启动后,玩家能够使用原来的角色进入
- [ ] 手动执行一次“立即备份”

## 附录:存档结构与手动迁移

一般情况下使用上面的内置流程即可;本节供需要了解原理或手动处理特殊情况的用户参考。

```
<服务器目录>/Pal/Saved/
├─ SaveGames/0/<世界 GUID>/          ← 每个世界对应一个文件夹
│  ├─ Level.sav                      世界本体(建筑、帕鲁、地形变化)
│  ├─ LevelMeta.sav
│  ├─ WorldOption.sav                创建世界时的设置快照
│  └─ Players/<PlayerUid>.sav        每位玩家的角色
└─ Config/WindowsServer/
   ├─ PalWorldSettings.ini           世界设置(由 GUI 生成,请勿手动修改)
   └─ GameUserSettings.ini           ★ DedicatedServerName 决定加载哪个世界
```

**最重要的一点**:服务器不会自动加载“唯一的那个”世界文件夹,而是读取
`GameUserSettings.ini` 中的 `DedicatedServerName=` 并查找对应 GUID 的文件夹。
手动迁移失败通常是这里没有对应正确。服务器找不到目标世界时,会直接创建一个新世界。
在 GUI 的“存档备份”标签页点击“设为当前世界”,即可自动修改这一项。

**手动迁移世界**:将整个 `<世界 GUID>` 文件夹放入目标实例的 `SaveGames/0/` 目录
(可以使用“模组”标签页中的文件管理器上传),然后点击“设为当前世界”。
**不要**直接复制旧的 `GameUserSettings.ini`,其中包含旧主机的 `PublicIP`、
`PublicPort` 和 RCON 设置。迁移只需要 `DedicatedServerName` 这一项,GUI 会自动修改。

**手动修复主机角色**:内置的“修复主机角色”功能与社区工具
[palworld-host-save-fix](https://github.com/xNul/palworld-host-save-fix) 的主要流程
(`guild_fix=False`)相同。需要手动执行时,请按照该工具的说明操作,并提前备份。

## 常见问题

- **导入后进入的是新世界?** 检查“存档备份”标签页中的“当前世界”标记是否位于你的世界上。
  内置导入流程会自动设置;手动迁移时需要自行设置。
- **主机以外的玩家也需要修复吗?** 不需要。联机存档中的其他玩家与专用服务器使用相同类型的 ID,
  可以直接使用原角色进入;只有绑定通用 ID 的主机角色需要迁移。
- **修复失败怎么办?** 修复前会强制自动备份。选择“存档备份”标签页最上方的备份即可一键恢复。
